# 시티즌 개발자 프로그램의 참가자 선발 기준·절차 모범 사례

## 핵심 발견

- **파일럿 부서 선정**: 강한 관심을 표명하고, 명확한 자동화/개발 백로그가 있으며, 내부 챔피언(champion)이 적어도 1명 이상인 2~3개 비즈니스 유닛을 파일럿 대상으로 우선 선정한다. ([Kissflow - Citizen Developer Program](https://kissflow.com/no-code/citizen-developer-program/), 날짜 미상)

- **팀 구성 원칙 (해커톤·파일럿 공통)**: 팀 규모는 3~6명(최적 3~5명)이 권장되며, 최소 1명의 비엔지니어(도메인 전문가·디자이너·운영 담당 등) 포함이 필수다. 전사 기술직+비기술직 혼합 구성이 가장 강력한 결과를 낸다. ([Innovation Mode - Corporate Hackathon Template](https://theinnovationmode.com/corporate-hackathon-template), 날짜 미상)

- **참가자 자격 기준**: IT 비전공자를 명시적으로 환영하며, 스프레드시트 등 기존 도구를 익숙하게 다루고, 업무 프로세스 전문성을 보유하며, 노코드/로우코드 앱 개발을 배우려는 의지가 있는 직원이 이상적인 시티즌 개발자 후보다. ([UiPath - What is Citizen Development](https://www.uipath.com/rpa/what-is-citizen-development), 날짜 미상)

- **부서장 승인 절차**: 참가자가 직속 상사(부서장)의 승인을 받아 업무 시간의 일정 부분을 파일럿 과제에 할당할 수 있도록 하는 것이 모범 사례다. 경영진 후원(executive sponsorship) 확보가 프로그램 성공의 전제 조건으로 강조된다. ([CIO.com - 해커톤 성공 비밀](https://www.cio.com/article/3530262/%EC%9D%BC%ED%84%B0%EB%A5%BC-%ED%98%81%EC%8B%A0-%EB%86%80%EC%9D%B4%ED%84%B0%EB%A1%9C%C2%B7%C2%B7%C2%B7-%EC%84%B1%EA%B3%B5%EC%A0%81%EC%9D%B8-%ED%95%B4%EC%BB%A4%ED%86%A4%EC%9D%98.html), 날짜 미상)

- **과제 심사 기준 (8개 가중 항목)**: 문제 중요성, 전략 테마 부합도, 실현 가능성, 개념 효과성, 개발 용이성, 운영 단순성, 잠재적 영향력, 혁신 수준의 8개 가중 기준으로 과제를 평가한다. 심사는 1단계(도메인 전문가 폐쇄형 점수화) → 2단계(상위 5팀 경영진 패널 공개 발표)의 2단계 구조를 권장한다. ([Innovation Mode - Corporate Hackathon Template](https://theinnovationmode.com/corporate-hackathon-template), 날짜 미상)

- **과제 제출·검토 2단계**: UiPath Automation Hub 프레임워크 기준으로 과제는 비즈니스 검토(넓은 사용자에게 이익이 되는지 평가) → 기술 검토(코드 품질·확장 준비도 확인)의 2단계를 거쳐 CoE 또는 시티즌 개발자 승인으로 연결된다. 역할은 셀프 유저, 파워 유저, 비즈니스 검토자, 기술 검토자 4종으로 구분된다. ([UiPath Automation Hub - Citizen Developer Framework](https://docs.uipath.com/automation-hub/automation-cloud/latest/user-guide/understanding-the-citizen-developer-framework), 날짜 미상)

- **CoE(Center of Excellence) 규모**: 프로그램 표준 수립, 사용 사례 승인, 플랫폼 관리, 교육, 커뮤니티 운영을 담당하는 CoE는 통상 IT+비즈니스 전문가 2~5명으로 구성하고, 신규 과제 제출 포털을 운영한다. ([Kissflow - Citizen Developer Program](https://kissflow.com/no-code/citizen-developer-program/), 날짜 미상)

- **파일럿 타임라인**: 90일(약 3개월)이 파일럿 기간의 업계 표준이며, 부서당 최대 5개의 사용 사례로 범위를 제한하는 것이 권장된다. ([Kissflow - Citizen Developer Program](https://kissflow.com/no-code/citizen-developer-program/), 날짜 미상)

- **LG유플러스 국내 사례**: 국내 최초 RPA 시민개발자 모델을 도입하여 탑다운(핵심 과제 전담 개발자 육성)·바텀업(전 직원 자동화 과제 개발 장려) 이원화 방식을 운영했다. 선발된 인원은 초급·중급·고급·특급 4등급으로 분류하여 등급별 혜택을 부여, 자발적 역량 향상을 유도했다. ([UiPath - LG유플러스 시민개발자 사례](https://www.uipath.com/ko/resources/automation-case-studies/lguplus), 날짜 미상)

- **Samsung C-Lab 사내벤처 사례**: 삼성전자 C-Lab Inside는 사내 해커톤 등을 통해 과제를 공모·선정하며, 선정된 과제는 최대 1년간 C-Lab 내에서 전담 지원을 받고 우수 과제는 스핀오프 창업으로 연결된다. 총 259개 프로젝트, 1,060명 참여 실적(2020년 기준). ([전자신문 - 5대 그룹 사내벤처 육성](https://m.etnews.com/20200319000047), 2020년 3월 19일)

## 한계

- 국내 대기업(SK, 현대 등)의 사내 시티즌 개발자/AI 코딩 파일럿 프로그램 내 **구체적인 지원 자격 요건(직급, 경력 연수, 사전 기술 수준 등)** 공개 자료를 확인하지 못함.
- **부서장 승인 절차의 세부 단계**(승인 양식, 시간 확보 비율, 승인 기간 등)를 정량적으로 제시한 1차 출처 미확보.
- 해커톤·파일럿 관련 수치 데이터의 상당수가 플랫폼 벤더(UiPath, ServiceNow, Kissflow) 자료이며, 독립적 제3자 연구 데이터는 제한적임.
- Claude Code 등 AI 코딩 도구를 활용한 시티즌 개발자 파일럿 선발 사례는 공개된 1차 출처를 찾지 못함.
- Gartner 원문 리포트는 유료 구독 필요로 인해 직접 확인 불가, 2차 인용 수치에 의존.
