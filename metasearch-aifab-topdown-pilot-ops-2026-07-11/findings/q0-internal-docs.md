## 핵심 발견

- 탑다운 과제는 사업 계획으로 확정된 공식 과제이며, 수행 주체는 "지정된 개발·운영 조직"으로 명시됨. 바텀업의 "시티즌 개발자(교육 이수자)"와 명확히 구분됨 — **이 문서는 시티즌 개발자 선발·파일럿 운영 시나리오를 직접 다루지 않음** (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §2. 바텀업 샌드박스와의 차이)
- 배포 승인 게이트는 스테이징 배포·검증 통과 후 "AI Board·정보보호팀 승인 게이트"(수동) → 무중단 배포(Blue/Green 또는 Rolling), 실패 시 자동 롤백 순서로 구성됨 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §5. 개발·배포)
- 운영 환경 직접 수정 경로는 없으며, 모든 변경은 Git 머지 → 파이프라인 재통과로만 반영 (불변 배포 원칙) (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §5. 개발·배포)
- 바텀업 우수과제 격상은 분기 보드 심의로 확정되며, 7단계 체크리스트(코드 리뷰 → 문서화 → 실데이터 재설계 → 운영 조직 지정 → 환경 재프로비저닝 → 이관·검증 → 원 환경 정리)로 이관 처리됨 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §6. 격상 유입 절차)
- 격상의 실체는 "재프로비저닝 + 재배포"이며 애플리케이션 재작성 없이 수용하는 것이 핵심 원칙으로 명시됨 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §6. 격상 유입 절차)
- SLO(가용성 예: 99.5%, 오류율, 응답시간)를 과제별로 보드가 확정하며, 주간 리포트로 보드·경영진에 보고 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §7. 운영·모니터링)
- 과제 종료·축소는 사업 판단(보드 심의)으로 결정하고, 종료 시 데이터 파기·자원 회수 체크리스트 적용. 자동 선셋은 적용하지 않음 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §7. 운영·모니터링)
- CI/CD 파이프라인 통과 기준: 단위 테스트 통과·하드코딩 시크릿 미검출(자동), Critical/High 취약점 0건(자동), 기능·성능 검증·정책 준수(자동+오너), 그 후 수동 승인 게이트 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §5. 개발·배포)
- 실데이터 취급 통제: 데이터 등급 분류 → 정보보호팀 사전 심사·승인 → 원본 복사 최소화 → KMS 암호화 → CloudTrail 전수 기록 → Macie 상시 탐지 → 종료 시 파기 확인 (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §4.1 실데이터 취급)
- 구축 로드맵은 M1~M6 4단계: 공통 기반(M1~M2) → 탑다운 표준화(M2~M3) → 배포 체계(M3~M4) → 격상 수용·정착(M4~M6) (내부: AWS_기반_탑다운_AIFAB_과제_운영환경_구축운영_기획안_v1-0.md §8. 단계별 구축 로드맵)

## 한계

- **브리프 핵심 질문 1(시티즌 개발자 선발 기준·절차)**: 본 문서는 탑다운 운영환경 구축·운영 기획안이며, 수행 주체를 "지정된 개발·운영 조직"으로 전제함. 시티즌 개발자 선발 기준·절차에 대한 내용 없음.
- **브리프 핵심 질문 2(3주 스프린트 주차별 운영 구조)**: 본 문서에 3주 스프린트 구조(킥오프, 중간 점검, 심사·게이트 일정)가 존재하지 않음. 파일럿 스프린트 운영 시나리오는 다루지 않음.
- **브리프 핵심 질문 3(멘토링·오피스아워·AI 코딩 도구 지원)**: 개발자 지원 체계(멘토링, 오피스아워, AI 코딩 도구)에 대한 내용 없음. 인프라·플랫폼 지원(Landing Zone, IaC, CI/CD)만 기술됨.
- **브리프 핵심 질문 5(파일럿 성공 KPI와 확대 판단 기준)**: 정량적 KPI 및 프로그램 확대 판단 기준 없음. SLO(가용성 99.5% 등)는 개별 과제 운영 수준 목표이며 파일럿 프로그램 KPI는 아님.
- 관련 문서로 언급된 "샌드박스 기획안(바텀업, v2-1)"은 분석 목록에 없어 내용 미확인. 시티즌 개발자 선발·스프린트 구조 관련 내용이 해당 문서에 있을 가능성 있음.
- 국내외 유사 프로그램 사례는 본 문서에 포함되지 않음.
