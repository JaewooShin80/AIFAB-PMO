## 1. 프로그램 개요

### 1.1 목적

시티즌 개발자 2~3팀(팀당 3~5명)을 선발해 **3주 스프린트로 T2Y 에이전트를 운영환경 배포까지 완성**하는 탑다운 파일럿을 9월 첫째 주에 개시한다. 파일럿으로 ① 비전문 개발자의 3주 운영 배포 가능성 ② 거버넌스 게이트 체계의 실효성 ③ 확대(2차 코호트) 타당성을 검증한다. 파일럿은 1회성 행사가 아니라 **지속 T2Y 에이전트 제작 사이클의 #1**이며, 판정 결과에 따라 4주 사이클(3주 스프린트 + 1주 판정·이관)로 정례화한다.

### 1.2 설계 원칙

- **플랫폼 선행**: 배포 자동화(골든 패스) 완비가 킥오프 전제 — 미완비 시 킥오프 순연 (일정보다 전제 충족 우선)
- **기준 사전 동결**: KPI·Go/No-Go 스코어카드를 킥오프 전(8/28) 서명 동결 — 사후 기준 변경 차단
- **게이트 통제**: Agile-Stage-Gate 하이브리드 — Gate 0(착수) → Gate 1(중간 9/11, 판정 SLA 48h) → Gate 2(배포 승인 9/16~17)
- **확대 대비 설계**: 파일럿은 2~3팀이나 플랫폼·비용 구조는 5~10팀 수용 기준으로 설계

## 6. 품질·보안 통제

1. **기술 내재화**: SAST(Semgrep)·시크릿 검출(gitleaks)·컨테이너 스캔(Inspector)을 파이프라인에 내장 — 통과 없이 배포 물리적 불가
2. **위험 등급별 리뷰(3-lane)**: Green 피어 1인 4h / Yellow 시니어 24h / Red(인증·개인정보·인프라) 3인 승인 24~48h + rubber-stamping 방지 장치
3. **최종 승인 게이트**: AI Board·정보보호팀 수동 승인 → Blue/Green 무중단 배포·자동 롤백
4. **AI 거버넌스**: Bedrock 호출 전량 로깅, Guardrails(민감정보 필터), 승인 모델 화이트리스트, 팀별 토큰 상한
5. **협의 관리**: 정보보호팀 협의 22건(기본안 병기)을 단일 문서로 관리 — 차단성 항목 8/28 확정: `AIFAB_AI인프라팀_정보보호팀_협의사항_최종_v1-0.md`
